# Quality Gate — PPT Pro Studio

渲染后逐条核对。任一不达标即退回 ⑧ 精修，禁止带缺陷交付。

## 程序化校验（MCP `qa_check` / 脚本）

- [ ] 页数 == brief.slides 数量
- [ ] 无空页（shapes 数为 0 或仅背景）
- [ ] 无占位符残留：`[图片缺失`, `TODO`, `xxx`, `Lorem`
- [ ] 无未替换模板变量：`{{...}}`, `<...>`

## 视觉 / 排版

- [ ] 文字无溢出文本框（单页正文框高度内可容纳，长文自动换行不裁切）
- [ ] 元素无重叠（标题栏与内容区、卡片间距 ≥ 0.1in）
- [ ] 每页 ≤ 3 核心要点
- [ ] 中文显示正常（非方块 / 非乱码）

## 设计一致性

- [ ] 配色统一，全程只用所选 style 的 palette
- [ ] 字体统一（中文 YaHei）
- [ ] 页码 / 页脚一致（封面、章节页除外）
- [ ] 品牌色正确（客户 VI 已应用）

## 内容质量

- [ ] 图表数据与文案一致（无矛盾数字）
- [ ] 数据真实或可溯源（标注假设时不冒充事实）
- [ ] 逻辑连贯：封面→目录→章节→内容→总结→结尾 闭环
- [ ] 受众匹配（学员版不堆术语，投资人版有 ROI）

## 商用 / 交付合规

- [ ] 输出为原生 `.pptx`，PowerPoint / WPS 中逐元素可编辑
- [ ] 无水印、无外部服务水印
- [ ] 可离线打开（无运行时网络依赖）
- [ ] 许可清晰：本技能 MIT，组件均宽松许可，可商用 / 再分发 / 收费
- [ ] 不侵犯第三方版权（图片用自产 / 授权素材）

## 退回规则

任一 [ ] 未勾选 → 定位到对应 slide 的 brief 字段 → 修正 brief.json → 重渲染。
**禁止**直接手改 .pptx 后再丢失源 brief（源丢失则无法迭代）。
