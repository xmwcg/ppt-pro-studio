## 二、关键发现：Marp PPTX 默认是「图片型」，正好印证你的路由设计

调研确认（marp-cli 官方文档）：
- `marp xxx.md --pptx` 默认导出**图片型 PPTX**（每页一张渲染图，不可逐元素编辑）。
- `--pptx-editable` 是实验性功能，且需额外装 LibreOffice Impress，**复杂样式难保真**。

这证明你的直觉完全正确，路由必须分两条：

| 用户意图 | 走哪条路径 | 产物 | 可编辑性 |
|---|---|---|---|
| 「可编辑 / 二次修改 / WPS 改 / 交付同事」 | **核心引擎**（ppt-master + python-pptx） | 原生 DrawingML .pptx | ✅ 逐元素可改 |
| 「震撼 / 视觉 / 演示天花板 / 吸引付费 / 路演」且无编辑诉求 | **Marp HTML 路径** | 高颜值 HTML PPT（+ 可选图片型 PPTX/PDF 分发） | HTML 可交互、PPTX 仅分发 |

**结论**：核心引擎负责「可交付的生产件」，Marp 路径负责「收割眼球的展示件」，二者同源（同一份结构化 brief）→ 不重复劳动。

## 三、路由逻辑（决策核心）

`brief.json` 新增字段 `delivery`：

- `"editable"`（**默认**）→ 核心引擎（ppt-master / python-pptx）
- `"showcase"` → Marp HTML 路径（演示天花板）

**提示词增强阶段自动识别意图**（无需用户显式填）：
- 命中 `可编辑|编辑|二次修改|WPS改|交付同事|可改` → `editable`
- 命中 `震撼|视觉|演示|吸引|付费展示|路演| showcace|天花板` 且无编辑诉求 → `showcase`
- 兜底默认 `editable`（安全侧，永远给可编辑生产件）

两条路径共享：Design System 2.0 token、Theme Market 调色板、QA 门禁（QA 2.1 扩到 HTML 维度）。

## 四、P3+ 精修清单（按优先级）

### P0 — 商业吸引力缺口，风险低、拉动最大

- **P3-1 升级 bundled ppt-master → v2.11**
  - 上游已大改：多画布（小红书 / 公众号 / A4）、图标库更新、稳定性。我们 vendor 可能落后。
  - *Why*：补「分发到短视频 / 公众号」场景，恰是知识付费增量。
  - *风险*：上游 SVG schema 可能变 → 升级后全量回归双引擎 QA。

- **P3-2 ① 提示词增强加「叙事结构 + 受众/目标」选择器**（借鉴 Githubfans518/PPT-Skill）
  - 4 种叙事骨架自动套：说服型 / 通知型 / 教学型 / 激励型。
  - 加受众（投资人 / 学员 / 老板 / 客户）+ 目标（转化 / 理解 / 决策）维度。
  - *Why*：直接拉高成稿质量与商业说服力，是云工具核心差异点。

- **P3-8 新增 Marp HTML 路径（演示天花板）** ← 本次新增
  - 新增 `scripts/ppt_marp.py`：brief → Marp Markdown（front-matter 注入主题 CSS + 每页 `notes`）→ `marp --html` 调 CLI。
  - 内置「showcase 主题包」：`references/marp-themes/` 下 3–4 套高视觉 CSS（玻璃拟态 / 渐变光晕 / 全屏大图 / 动态背景 keyframes），把 HTML 输出顶到视觉天花板。
  - 一键出三格式：`xxx.html`（默认，视觉天花板）+ `xxx.pdf`（归档）+ `xxx.pptx`（图片型分发，不强求编辑）。
  - *依赖*：本地装 `@marp-team/marp-cli`（Node 18+）；PPTX/PDF 需 Chrome。提供 standalone binary 选项免 Node。
  - *Why*：用最低成本补齐「视觉吸引力」，直接服务付费转化卖点。

### P1 — 交付零翻车 + 补齐展示/市场

- **P3-3 QA 2.0 → 2.1**
  - 新增审计维度：中文字体缺失、WCAG 对比度、截断验证、图片分辨率/占比；HTML 路径加「动画/过渡/移动端响应式」检查。

- **P3-9 模板市场（Template Market）** ← 本次新增
  - 在 `theme_market.py` 基础上扩为 `template_market.py`：模板 = 主题 + 版式组合 + 示例内容 + 配图位 + 推荐叙事结构。
  - 起步内置 10+ 套行业模板（融资路演 / 课程课件 / 年度复盘 / 产品发布 / 小红书知识卡 / 党政汇报 …）。
  - CLI：`list / show / validate / init`；MCP `list_templates`。
  - *Why*：模板可售源文件 = 被动收入（参考调研：单模板 19–399 元）；且降低用户门槛。

- **P3-10 UI 控制台（本地向导）** ← 本次新增
  - 纯本地 `ui/console.html`（HTML/JS）：填表 → 生成 `brief.json` → 选引擎/路径 → 实时预览 SVG/HTML → 一键生成。
  - 含「模板市场」可视化挑选、「主题色卡」实时预览、「叙事结构」下拉。
  - *Why*：对标云工具「控制台体验」，把本地 Skill 的门槛降到最低。

## 五、商业变现闭环

```
结构化 brief（一套源）
   ├─ editable 路径 → 核心引擎 → 原生可编辑 .pptx  （交付生产件，客户可改）
   └─ showcase 路径 → Marp → 高颜值 HTML/PDF/PPTX   （收割眼球，吸引付费）
                                          │
                            模板市场（可售源文件）
                                          │
                            导入乐享/腾讯文档（协作+分发）
                                          ▼
                              知识付费 / 模板销售 / 定制服务
```

卖点排序（对外话术）：
1. **真可编辑**（vs 多数网页 PPT 导出即图片）→ 核心引擎兜底。
2. **视觉天花板**（vs 普通可编辑 PPT 丑）→ Marp 展示路径。
3. **全离线可商用 MIT**（vs 订阅+隐私风险）。
4. **模板市场 + 控制台**（vs 命令行门槛）。
5. **双格式一套源**（一份 brief 同时出生产件 + 展示件）。
